# Security Policy

## Reporting a Vulnerability

Please do not open public issues for security problems affecting this site,
newsletter forms, comments, deployment configuration, or accidental disclosure of
private content. Use GitHub private vulnerability reporting if it is available,
or email the maintainer listed on the profile.

Include:

- A short description of the issue.
- Reproduction steps or a minimal proof of concept.
- The affected page, form, dependency, or deployment setting.
- Whether private data, tokens, or unpublished content may be involved.
